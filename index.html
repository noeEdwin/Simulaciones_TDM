Conceptos Clave

Multiplexación por División de Tiempo Síncrona

Es la forma clásica de TDM. El multiplexor asigna una ranura de tiempo (time slot) fija a cada dispositivo conectado, tenga datos para enviar o no.

Ventajas

  • Simple de implementar.
  • Garantiza ancho de banda a cada usuario.
  • No requiere "direccionamiento" (el orden indica el dueño).

Desventajas

  • Ineficiente: Si un usuario no transmite, su espacio se desperdicia (envía espacios vacíos).

Multiplexación Estadística (Asíncrona)

El ancho de banda se asigna bajo demanda. Solo los dispositivos que tienen algo que enviar ocupan las ranuras de tiempo.

Ventajas

  • Muy eficiente: No se desperdicia tiempo en canales inactivos.
  • Soporta más usuarios conectados que ranuras disponibles.

Desventajas

  • Complejidad: Requiere buffers (memoria) y direccionamiento.
  • Cada paquete debe llevar la dirección de destino (Overhead).
